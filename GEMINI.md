# BestWay プロジェクト定義書

## 1. コンセプト
**「何がわからないかわからない」を整理し、AIと人間の間を翻訳する“理解支援OS”**
単なるチャットツールではなく、ユーザーの状況・理解度・目的を整理し、最適な学習ロードマップと質問内容を生成することで、学習者やIT初心者をゴールまで導く。

## 2. 開発の動機・経緯（誕生の物語）

### 「できる人」と「できない人」の差
当初からこの差に興味を持っていた。

### “学習”への興味と不安
勉強の最初が難しいと痛感。エンジニア用語（フロントエンド、バックエンド、HTML/JS/Python等）を調べても全体像が見えず、どこがゴールで何をどの順番で学ぶべきか、時間がかかるかも不明で、勉強法が正しいかも分からない不安を抱えていた。

### SG試験・ITパスポートの気づき
参考書の深読みではなく「頻出・ポイント・過去問」という「やり方」が重要だと気づき、「勉強内容」より「勉強のやり方」を教えることが大事ではないかと思い始める。

### 学習OSという考え
そこから「勉強そのもの」ではなく「学び方」を整理するアプリを考案。フェーズやロードマップの可視化、今どこにいて何が足りないかを示す仕組みを構想した。

### “質問できない問題”への気づき
おじいちゃんの「iCloud容量問題」との対話が転換点。相手は現状しか言えないが、本当の悩みは背景に隠れている。AIはすぐに解決策を提示するが、本当に必要なのは「まず何に困っているかを整理すること」だと気づき、「質問できない」という問題の深さを認識した。

### AIの限界への気づき
AIは賢いが「人間がうまく聞けない」問題がある。「PHPってバックエンド？」という質問の裏にある背景理解（フロント/バックの違い、Web全体像、言語の役割）までAIは読み取れない。そこで「AIの前に、人間側を整理する」という発想に至った。

### BestWay 誕生
“何がわからないかわからない”を整理するAIへ進化。「AIと人間の間を翻訳する」アプリを目指す。

## 3. BestWay の核
- **本質**: 「答えを出すこと」ではなく「状況を理解すること」。
- **テーマ**: 「質問できない人を助ける」。

## 4. 今後の方向性
- **応用の可能性**: 学習支援（IT・英語・資格等）、高齢者DX支援（設定・スマホの使い方）、問い合わせ整理など。
- **技術の具体化**: localLLM や OpenAI API の使い分け、Memoryシステムによる文脈保持、動的なロードマップ生成など現実的な構成を検討中。

## 5. 開発・学習戦略
- **実践主義**: 教本を深追いせず「作りたいもの」のために必要な技術を都度学び、実装する。
- **将来への投資**: Python (AI/バックエンド) ＋ TypeScript (フロントエンド/Next.js) という市場価値の高いスタックを選択。
- **アジャイル**: 完璧主義を捨て、小さな動くデモを積み重ねて成長させる。

## 6. 技術スタック
- **Frontend**: Next.js (TypeScript) - 初心者UXの追求
- **Backend**: FastAPI (Python) - AIエージェント / APIサーバー
- **Database**: Supabase (PostgreSQL)
- **管理**: Git / GitHub
